Collect Feedback

Five ways feedback arrives. Each normalizes into the same schema, runs auto-QA, and lands on the Review Board.

Paste a shared Google Sheet / Doc URL, plus an optional Drive folder URL. In a connected workspace this is pulled through a Google Drive connector; demo data is shown here.

Import the customer's spreadsheet. CSV parses fully offline; XLSX loads the SheetJS parser on first use (needs internet for that step). Columns auto-map by header (Scope, Source English, Current German, Proposed German fix, Issue Type, ...).

Attach customer screenshots and paste the chat / message they sent. The screenshot rides along as evidence; the text is parsed into a draft feedback item (full AI extraction in production).

Paste a customer Jira ticket (JSON or key:value lines). We map it into a LOC localization ticket and add it to the board. In production this creates a real ticket in the LOC Jira project via the Atlassian connector (with your confirmation).

Five ways feedback comes in

  • Manual entry — the structured form.
  • Google Doc / Drive URL — paste a shared link; in a connected workspace it is pulled via the Google Drive connector (demo data shown here).
  • Import XLSX / CSV — drop the customer's spreadsheet; columns auto-map. CSV is fully offline; XLSX uses the SheetJS parser (loaded on first use).
  • Screenshots + chat — attach images (embedded as evidence) and paste the message; the text is parsed into a draft.
  • Jira ticket -> LOC — paste a customer ticket; we generate a LOC localization ticket with mapped fields.

Data provenance

Seeded with 23 representative issues (8 RingEX + 15 RingCX): English source, current German, and proposed fixes.

This is the public demo build: internal product screenshots, audio, reporter names and internal system URLs have been removed. Evidence slots show representative placeholders. The internal version (with the real evidence attached) is kept private.

How the auto-QA verdict is computed

Each proposed German fix is checked against the RC de-DE glossary (3,173 terms) and the DE-DE style guide (131 rules) - the same references used by the RC translation-QA toolchain.

Passglossary-aligned / clearly correct
Reviewdefensible but a terminology decision is needed
Conflictfix contradicts an approved glossary term
Needs contextsource string ambiguous - insufficient info
Gapno fix was provided - QA proposes one

Confidence scores are advisory; a human linguist makes the final call. Where the source is unclear, the QA says so rather than guessing.

Demo limitations & next steps

  • Your board is saved in this browser, so a refresh keeps your data. Use Start fresh (top bar) to begin an empty collection. Export JSON/CSV for backup / handoff.
  • The Google-Doc and Jira modes show the ingest + field-mapping; live fetching/ticket-creation runs server-side via the connectors (the Drive pull was already proven real).
  • Auto-QA is pre-computed for the seed set and heuristic for new items; production runs the full RC translation-QA engine on every submission.
